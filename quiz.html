Level 1 -- Foundations
Dummy Head Node
When the head itself might be removed or changed
▶
Sentinel / Dummy Node
Add a fake node before head so you never have to special-case head deletion/insertion. Return dummy.next at the end.
Reverse a Linked List (LC #206)
Reverse all nodes -- O(1) space, single pass
▶
Iterative Reversal Three Pointers (prev/curr/next)
The prev/curr/next pointer dance: save next, point curr back to prev, advance both. The fundamental building block for all reversal problems.
Linked List Cycle (LC #141)
Determine if a cycle exists -- O(1) space
▶
Slow/Fast Pointers Floyd's Algorithm
Slow moves 1, fast moves 2. If there's a cycle, they must meet. If fast hits null, no cycle. The O(1) space constraint is the giveaway.
Level 2 -- Applied Foundations
Middle of the Linked List (LC #876)
Find the middle node in a single pass -- no length known
▶
Slow/Fast Pointers
When fast reaches the end, slow is at the middle. "Single pass, no length" screams slow/fast.
Remove N-th Node From End (LC #19)
Remove the n-th node from the end in one pass
▶
Two-Pointer Gap Dummy Head
Advance fast pointer n steps ahead, then move both. When fast hits null, slow is right before the target. Dummy head handles edge case of removing the first node.
Linked List Cycle II (LC #142)
Find the node where the cycle begins
▶
Floyd's Cycle Detection Slow/Fast + Reset
After slow & fast meet, reset one to head and walk both at speed 1. They meet at the cycle entry point. Mathematical proof: distance from head = distance from meeting point to entry.
Merge Two Sorted Lists (LC #21)
Merge two sorted linked lists into one sorted list
▶
Dummy Head Comparison Walk
Dummy head avoids "which list has the smaller first node?" edge case. Then just compare and append the smaller node each step.
Remove Duplicates (LC #83 / #82)
Delete duplicate values from a sorted list
▶
Pointer Skip Dummy Head (#82)
#83: If curr.val == curr.next.val, skip next. #82 (remove ALL duplicates): needs dummy head because head itself might be a duplicate. Sorted order is the key enabler.
Level 3 -- Multi-Technique Combos
Reverse Linked List II (LC #92)
Reverse only nodes from position left to right
▶
Partial Reversal Dummy Head Three Pointers
Walk to position left-1 (the "before" node), then do the prev/curr/next reversal for (right-left) steps. Dummy head handles left=1 edge case.
Swap Nodes in Pairs (LC #24)
Swap every two adjacent nodes (not just values)
▶
Dummy Head Iterative Pair Swap
Dummy head + rewire in pairs. It's K-group reversal where K=2. Keep a "prev" pointer to reconnect each swapped pair to the rest of the list.
Palindrome Linked List (LC #234)
Check if a linked list is a palindrome -- O(1) space
▶
Slow/Fast (Find Middle) Reversal Two-Pointer Compare
Three patterns combined: 1) Find middle with slow/fast, 2) Reverse second half, 3) Compare both halves node by node. The O(1) space constraint rules out using an array.
Sort List (LC #148)
Sort a linked list in O(n log n) time, O(1) space
▶
Merge Sort Slow/Fast (Find Middle) Merge Two Sorted
O(n log n) + linked list = merge sort. Split with slow/fast, recursively sort halves, merge with the comparison walk. Linked lists make merge sort natural because splitting is O(1) pointer work.
Partition List (LC #86)
Rearrange so all nodes < x come before nodes >= x
▶
Two Dummy Heads Split & Reconnect
Build two separate lists (less-than and greater-or-equal), each with its own dummy head. Walk the original, append to the right list. Then connect the tails.
Reorder List (LC #143)
Reorder L0 -> Ln -> L1 -> Ln-1 -> ...
▶
Slow/Fast (Find Middle) Reversal Merge / Interleave
1) Find middle with slow/fast, 2) Reverse second half, 3) Interleave the two halves. Very similar to palindrome check but with a merge step instead of compare.
Level 4 -- Advanced Combos
Reverse Nodes in K-Group (LC #25)
Reverse nodes in groups of k -- leave remainder as-is
▶
Dummy Head Count + Check Sublist Reversal
For each group: count k nodes ahead (if fewer, stop). Reverse the sublist, reconnect to previous group's tail. Dummy head handles the first group cleanly.
Flatten a Multilevel Doubly Linked List (LC #430)
Flatten child pointers into a single-level list
▶
DFS / Stack Pointer Rewiring
When you hit a child, push next onto a stack (or recurse), splice the child list in. "Multilevel" = tree-like structure = DFS.
Copy List with Random Pointer (LC #138)
Deep copy a list where each node has a random pointer
▶
Hashmap (old -> new) Interleave Technique
Hashmap approach: Map each original node to its clone, then wire up next/random from the map. O(1) space trick: interleave clones between originals (A->A'->B->B'), then split.
LRU Cache (LC #146)
O(1) get and put with eviction of least recently used
▶
Doubly Linked List Hashmap Two Dummy Nodes (head/tail)
Hashmap gives O(1) lookup. Doubly linked list gives O(1) remove/insert for recency ordering. Two sentinel nodes (head & tail dummies) eliminate all edge cases in the list operations.
