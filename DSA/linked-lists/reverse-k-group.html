Reverse Nodes in K-Group
LC #25 | The Boss Fight
🔴 The Problem
Given a linked list, reverse every k consecutive nodes.
If remaining nodes < k, leave them as-is.
Input: [1] → [2] → [3] → [4] → [5], k = 3

🧠 Strategy
1. Use dummy node before head.
2. For each group: check if k nodes exist ahead.
3. If yes, reverse the k nodes (standard iterative reversal).
4. Reconnect: groupPrev → new head, old head (now tail) → next group.
5. Advance groupPrev to the tail of the reversed group. Repeat.
📸 Walkthrough (k=3)
Check: do 3 nodes exist?
Walk k steps from groupPrev. If we reach null before k steps → not enough nodes → stop.

💻 Code (JavaScript)
function reverseKGroup(head, k) { const dummy = { next: head }; let groupPrev = dummy; while (true) { // Check if k nodes remain let check = groupPrev; for (let i = 0; i < k; i++) { check = check.next; if (check === null) return dummy.next; } // Reverse k nodes let prev = null; let curr = groupPrev.next; const groupTail = curr; // will be tail after reverse for (let i = 0; i < k; i++) { const next = curr.next; curr.next = prev; prev = curr; curr = next; } // Reconnect groupPrev.next = prev; // prev is new group head groupTail.next = curr; // curr is next group's first // Advance groupPrev = groupTail; } }
🔑 The Reconnection
After reversing k nodes:
prev = last reversed node (new group head)
curr = first node of next group (or null)
groupTail = first node before reversal (now last)

Two reconnections:
groupPrev.next = prev (previous section → new head)
groupTail.next = curr (new tail → next section)
Time: O(n)
Space: O(1)
💡 Key Insights
1. This is Swap Pairs generalized — instead of reversing 2 nodes, reverse k. Same outer loop pattern.
2. The "check first" step prevents reversing a partial group — walk k nodes ahead before doing anything.
3. The inner reversal is the exact same prev/curr/next dance from "Reverse a Linked List" — just limited to k iterations.
4. Everything from Levels 1-3 converges here: dummy head, iterative reversal, pointer tracking.
