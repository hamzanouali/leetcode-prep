Palindrome Check
LC #234 | Find Middle + Reverse + Compare
🔴 The Problem
Given the head of a singly linked list, determine if it is a palindrome.
Do it in O(1) space (no array copy).
Example: [1] → [2] → [3] → [2] → [1] → null (palindrome!)

🧠 Strategy — 3 Steps
Step 1: Find the middle (slow/fast pointers).
Step 2: Reverse the second half.
Step 3: Compare first half with reversed second half.

Step 3: Compare
Walk both halves. Compare values node by node.

💻 Code (JavaScript)
function isPalindrome(head) { // Step 1: Find middle let slow = head, fast = head; while (fast !== null && fast.next !== null) { slow = slow.next; fast = fast.next.next; } // Step 2: Reverse second half let prev = null, curr = slow; while (curr !== null) { const next = curr.next; curr.next = prev; prev = curr; curr = next; } // Step 3: Compare halves let p1 = head, p2 = prev; while (p2 !== null) { if (p1.val !== p2.val) return false; p1 = p1.next; p2 = p2.next; } return true; }
⚠️ Even vs Odd Length
Odd (1→2→3→2→1): slow lands on 3. We reverse from 3 onward. The middle node is shared — doesn't matter since comparison walks until p2 hits null.

Even (1→2→2→1): slow lands on the 2nd "2". Reverse gives us [1→2] and [1→2]. Both halves match perfectly.
Time: O(n)
Space: O(1)
💡 Key Insights
1. This combines three Level 1 patterns: find middle + reverse + two-pointer compare.
2. It's destructive — modifies the list. In interviews, mention you could reverse back to restore the original list.
3. The O(n) space approach (copy to array, check palindrome) is easier but less impressive. This O(1) space version shows mastery.
