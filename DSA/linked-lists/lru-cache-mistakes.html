LRU Cache
Common Mistakes -- LC #146
Mistake #1: Using a singly linked list
The Wrong Approach
Use a singly linked list for the recency queue. When you need to remove a node from the middle (on cache hit), you can't get to its previous node in O(1). You end up walking the list, making get() O(n).
What should have tipped you off
"O(1) for both get and put." Removal from an arbitrary position in O(1) requires doubly linked -- you need to update both neighbors.
The Fix
Use a doubly linked list. Each node has prev and next. To remove: node.prev.next = node.next, node.next.prev = node.prev. O(1).
Mistake #2: Forgetting to update the hashmap
The Wrong Approach
On put() when the key already exists: update the node's value and move it to front, but forget this isn't a new insertion. Or worse: add a second node without removing the old one. Now the map points to a stale node.
What should have tipped you off
put(key, value) says "update the value if the key exists." That means put() has TWO branches: update vs insert. Each needs different logic.
The Fix
On put(): check if key exists first. If yes: update value, move to front, done. If no: create node, add to front, add to map, check capacity and evict if needed.
Mistake #3: Not using sentinel head/tail nodes
The Wrong Approach
Manage real head/tail pointers manually. Every insert and remove needs null checks: "Is the list empty? Is this the only node? Am I removing the head? The tail?" This creates 4-5 edge cases that are easy to get wrong under pressure.
What should have tipped you off
Any time you see yourself writing "if (head === null)" or "if (node === tail)" -- that's the signal you need sentinels.
The Fix
Create dummy head and dummy tail, linked to each other: head.next = tail, tail.prev = head. Now "add to front" always means insert after head. "Remove LRU" always means remove tail.prev. Zero null checks needed.
The Clean Implementation

Write just 2 helper functions: remove(node) -- unlinks a node from wherever it is. addToFront(node) -- inserts right after dummy head. Then get() = remove + addToFront + return value. put() = remove (if exists) + addToFront + maybe evict tail.prev. Everything else composes from these two.
