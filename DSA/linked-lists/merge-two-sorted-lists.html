LC #21 | Dummy Head + Comparison Walk
🔴 The Problem

Return the head of the merged list.

🧠 Strategy
1. Create a dummy node as the start of our merged list.
2. Use a tail pointer to track the end of the merged list.
3. Compare heads of both lists — attach the smaller one to tail.
4. When one list is exhausted, attach the remainder of the other.

💻 Code (JavaScript)
function mergeTwoLists(l1, l2) { const dummy = { next: null }; let tail = dummy; while (l1 !== null && l2 !== null) { if (l1.val <= l2.val) { tail.next = l1; l1 = l1.next; } else { tail.next = l2; l2 = l2.next; } tail = tail.next; } // Attach whatever remains tail.next = l1 !== null ? l1 : l2; return dummy.next; }
🛡️ Why Dummy Node?
Without dummy, we'd need special logic to decide which node becomes the head of the merged list. Dummy eliminates this — just return dummy.next.
Time: O(n + m)
Space: O(1)
💡 Key Insights
1. This is the merge step of merge sort — you'll reuse this exact logic in "Sort a Linked List" (LC #148).
2. The tail pointer pattern: always keep a reference to the last node of what you're building.
3. The remainder attachment (tail.next = l1 || l2) is key — no need to loop through the rest, just link it.
