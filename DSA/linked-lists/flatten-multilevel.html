Flatten Multilevel List
LC #430 | DFS / Stack Flattening
🔴 The Problem
A doubly linked list where nodes can have a child pointer to another list.
Flatten it so all nodes are in a single level, in DFS order.

🧠 Strategy — Iterative
Walk the list. When you find a node with a child:
1. Save next (the rest of the current level).
2. Set curr.next = child, child.prev = curr.
3. Walk to the tail of the child list.
4. Connect tail.next = saved next.
5. Clear the child pointer. Continue walking.
📸 Walkthrough
At node 3 (has child → 6)
Save next = node 4.
3.next = 6, 6.prev = 3.
Walk child to its tail (find tail of 6→7→8, but 7 has a child too!).
We'll handle 7's child when we get there naturally.
At node 7 (has child → 9)
Save next = node 8.
7.next = 9, 9.prev = 7.
Walk to tail of child: 9→10. Tail = 10.
10.next = 8, 8.prev = 10.
Back at child-list tail for node 3
The child list 6→7→9→10→8 has tail = 8.
8.next = 4, 4.prev = 8.

💻 Code (JavaScript)
function flatten(head) { let curr = head; while (curr !== null) { if (curr.child) { const next = curr.next; // save // Splice child in curr.next = curr.child; curr.child.prev = curr; curr.child = null; // Find tail of the child list let tail = curr.next; while (tail.next !== null) { tail = tail.next; } // Connect tail to saved next tail.next = next; if (next !== null) { next.prev = tail; } } curr = curr.next; } return head; }
📚 Alternative: Stack
When you hit a child: push next onto stack, follow the child.
When you hit null (end of a level): pop from stack, continue there.
Same result, more explicit DFS. O(depth) space for the stack.
Time: O(n)
Space: O(1)
Iterative approach is O(1) space. Stack approach is O(depth).
💡 Key Insights
1. The iterative approach handles nested children naturally — after splicing in a child, we continue walking. If the child list has its own children, we'll hit them.
2. Don't forget: clear the child pointer (curr.child = null) and set prev pointers (it's a doubly linked list).
3. The "find tail" walk makes worst case O(n) per child, but total work is still O(n) — each node visited at most twice.
