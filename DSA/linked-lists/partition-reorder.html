🔴 Problem 1 — Partition (LC #86)
Given head and value x, partition so all nodes < x come before nodes ≥ x. Preserve original order within each group.
Input: [1] → [4] → [3] → [2] → [5] → [2], x = 3
1. Create two dummy lists: less and greater.
2. Walk the original list, appending each node to the right bucket.
3. Connect less tail → greater head. Terminate greater with null.

greater: [4] → [3] → [5]

💻 Partition Code
function partition(head, x) { const lessD = { next: null }; const greaterD = { next: null }; let less = lessD, greater = greaterD; let curr = head; while (curr !== null) { if (curr.val < x) { less.next = curr; less = less.next; } else { greater.next = curr; greater = greater.next; } curr = curr.next; } greater.next = null; // terminate! less.next = greaterD.next; // connect return lessD.next; }
🔴 Problem 2 — Reorder (LC #143)
Reorder: L0 → Ln → L1 → Ln-1 → L2 → Ln-2 → ...
Interleave first half with reversed second half.
Input: [1] → [2] → [3] → [4] → [5]
1. Find middle (slow/fast).
2. Reverse second half.
3. Interleave (merge alternating nodes).

💻 Reorder Code
function reorderList(head) { if (!head || !head.next) return; // 1. Find middle let slow = head, fast = head; while (fast.next && fast.next.next) { slow = slow.next; fast = fast.next.next; } // 2. Reverse second half let prev = null, curr = slow.next; slow.next = null; // cut while (curr) { const next = curr.next; curr.next = prev; prev = curr; curr = next; } // 3. Interleave let p1 = head, p2 = prev; while (p2) { const tmp1 = p1.next, tmp2 = p2.next; p1.next = p2; p2.next = tmp1; p1 = tmp1; p2 = tmp2; } }
Time: O(n)
Space: O(1)
💡 Key Insights
1. Partition: the "two dummy lists" pattern — split into buckets, then reconnect. Don't forget greater.next = null! (avoids cycle).
2. Reorder: combines three techniques — find middle, reverse, interleave. Same building blocks as palindrome check.
3. The interleave step is the only new part — save next pointers before rewiring, advance both lists.
