Reverse Between L and R
LC #92 | v2: The "Classic Reversal + Reconnect" Approach
🔴 The Problem
Given a linked list, reverse nodes from position left to right (1-indexed).
Example: left=2, right=4

💡 Big Idea
Think of it as 3 separate zones:
Zone A — untouched prefix (before left)
Zone B — the sub-list to reverse
Zone C — untouched suffix (after right)

We just reverse Zone B exactly like a normal "reverse linked list", then stitch the 3 zones back together.
[Zone A] → [Zone B — reverse me] → [Zone C]
🧩 The 4 Pointers You Need
Save 4 pointers before reversing so you can reconnect after:

last node of Zone A
leftNode
first node of Zone B

📸 Walkthrough
Step 1: Navigate to find the 4 pointers
Walk (left - 1) steps → beforeLeft = node 1
leftNode = beforeLeft.next = node 2
Continue to position right → rightNode = node 4
afterRight = rightNode.next = node 5
Step 2: Standard reverse on Zone B
Reverse nodes 2→3→4 using the classic prev/curr/next technique you already know:
prev=null, curr=2: next=3, 2→null, prev=2, curr=3
prev=2, curr=3: next=4, 3→2, prev=3, curr=4
prev=3, curr=4: next=5, 4→3, prev=4, curr=5 ✓
Zone B reversed: 4 → 3 → 2
Step 3: Reconnect (just 2 links!)
1. beforeLeft.next = rightNode (Zone A → reversed head)
2. leftNode.next = afterRight (reversed tail → Zone C)

💻 Code (JavaScript)
function reverseBetween(head, left, right) { const dummy = { next: head }; let beforeLeft = dummy; // Step 1: find beforeLeft for (let i = 0; i < left - 1; i++) { beforeLeft = beforeLeft.next; } let leftNode = beforeLeft.next; // Step 2: classic reverse from leftNode // for (right - left) steps let prev = null, curr = leftNode; for (let i = 0; i <= right - left; i++) { const next = curr.next; curr.next = prev; prev = curr; curr = next; } // now prev = rightNode, curr = afterRight // Step 3: reconnect beforeLeft.next = prev; // Zone A → reversed head leftNode.next = curr; // reversed tail → Zone C return dummy.next; }
🔑 Why This Clicks
If you already know how to reverse a full linked list, this is the same thing — just on a sub-section.

The only new part is the reconnection step: stitching Zone A and Zone C back to the reversed chunk.

Mental model: cut the sub-list out → reverse it → tape it back in.
Time: O(n)
Space: O(1)
💡 Key Insights
1. This is literally "reverse linked list" + "save 4 pointers" + "2 reconnection lines".
2. After the reversal loop, prev = new head of reversed section, curr = first node after it. That's all you need to reconnect.
3. The dummy node handles the edge case where left=1 (reversing from the very start).
