Deep Copy with Random Pointers
LC #138 | HashMap or Interleave Clone
🔴 The Problem
Clone a linked list where each node has next and a random pointer (can point to any node or null).
Create a deep copy — entirely new nodes, same structure.
A → B → C → null
A.random → C
B.random → A
C.random → B
🧠 Approach 1 — HashMap (easier)
Pass 1: Create a clone of each node. Store oldNode → cloneNode in a Map.
Pass 2: Wire up next and random using the map to find clone nodes.
Pass 1: Create clones
map[A] = A', map[B] = B', map[C] = C'
(each clone has same val, but next and random are unset)
Pass 2: Wire pointers
A'.next = map[A.next] = B'
A'.random = map[A.random] = C'
...repeat for B' and C'
💻 HashMap Code
function copyRandomList(head) { if (!head) return null; const map = new Map(); // Pass 1: create clones let curr = head; while (curr) { map.set(curr, { val: curr.val, next: null, random: null }); curr = curr.next; } // Pass 2: wire pointers curr = head; while (curr) { const clone = map.get(curr); clone.next = map.get(curr.next) || null; clone.random = map.get(curr.random) || null; curr = curr.next; } return map.get(head); }
Time: O(n)
Space: O(n)
🧠 Approach 2 — Interleave (O(1) space)
Step 1: Insert clones right after originals: A→A'→B→B'→C→C'
Step 2: Set random: clone.random = original.random.next
Step 3: Separate into two lists.
Step 1: Interleave

original clone alternating
Step 2: Set random pointers
A.random = C → A'.random = C.next = C'
The clone is always right after the original!
Step 3: Un-interleave
Restore original list: A→B→C
Extract clones: A'→B'→C'
💻 Interleave Code
function copyRandomList(head) { if (!head) return null; let curr = head; // Step 1: Insert clones after originals while (curr) { const clone = { val: curr.val, next: curr.next, random: null }; curr.next = clone; curr = clone.next; } // Step 2: Set random pointers curr = head; while (curr) { if (curr.random) { curr.next.random = curr.random.next; } curr = curr.next.next; } // Step 3: Separate lists curr = head; const cloneHead = head.next; while (curr) { const clone = curr.next; curr.next = clone.next; clone.next = clone.next ? clone.next.next : null; curr = curr.next; } return cloneHead; }
Time: O(n)
Space: O(1)
💡 Key Insights
1. The core challenge: random can point anywhere — you can't set it until all clones exist. The map/interleave solve this.
2. HashMap is cleaner and easier to code. Use it in interviews unless asked for O(1) space.
3. Interleave is clever — by placing clones next to originals, original.random.next is always the corresponding clone.
4. Don't forget to restore the original list in the interleave approach!
