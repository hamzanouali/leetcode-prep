Pattern Triggers
How to read a problem and know which pattern to use
The 5 Core Patterns

When you see these keywords or constraints, reach for these patterns:

Dummy Head Node
"head might change" -- deletion/insertion at front
"merge two lists" -- need a neutral starting point
"remove nodes" where head could be the target
"partition" / "split into groups" -- one dummy per group
Problems: LC #21, #82, #86, #19, #24, #25, #92
Slow / Fast Pointers
"find middle" -- single pass, no length
"detect cycle" / "circular" -- O(1) space required
"k-th from end" -- gap technique variant
"palindrome" -- need middle to split and reverse
Problems: LC #876, #141, #142, #234, #148, #143
Iterative Reversal (prev/curr/next)
"reverse" anywhere in the problem name
"swap adjacent" / "in groups of k"
"reorder" -- often needs reversing second half
"palindrome check" -- reverse half, then compare
Problems: LC #206, #92, #24, #25, #234, #143
Two-Pointer Gap
"n-th from end" / "k-th from end" -- classic gap
"one pass" + need position relative to tail
"without knowing length" -- gap avoids counting
Problems: LC #19, #61
Hashmap + Linked List
"O(1) get and put" -- need both lookup + ordering
"deep copy" / "clone" with random pointers
"cache" / "LRU" / "LFU" -- map for speed, list for order
Problems: LC #146, #138, #460
Quick Decision Flowchart
Does the problem say "reverse"?
Yes -- Use prev/curr/next reversal. Add dummy head if reversing from position 1.
Does it mention cycle, middle, or palindrome?
Yes -- Use slow/fast pointers. Palindrome = slow/fast + reversal combo.
Does it say "k-th/n-th from end"?
Yes -- Use two-pointer gap + dummy head.
Could the head node change? (deletion, merge, partition)
Yes -- Use dummy head. Create 2 dummies for split/merge problems.
Need O(1) lookup + ordering? Random pointers? Cache?
Yes -- Use hashmap + linked list.
Pattern Combos

Harder problems combine multiple patterns. Learn to spot the recipe:

Palindrome Check (LC #234)
slow/fast (find middle) + reversal (flip second half) + compare
Sort List (LC #148)
slow/fast (split) + merge sort recursion + dummy head merge
Reorder List (LC #143)
slow/fast (find middle) + reversal (flip second half) + interleave
Reverse K-Group (LC #25)
count k + sublist reversal + dummy head reconnection loop
LRU Cache (LC #146)
hashmap (O(1) lookup) + doubly linked list + 2 dummy sentinels (head/tail)
The Real Interview Skill

Knowing the patterns is only half the battle. The other half is reading the constraints: O(1) space? Rules out hashmap -- think pointers. O(n log n)? Think merge sort. Single pass? Think two-pointer gap or slow/fast. "In-place"? Think reversal + pointer rewiring.

When You're Stuck

Ask yourself: "What's annoying about this problem?" If head might change -- dummy head. If you need to find a position without length -- slow/fast. If order needs to flip -- reversal. If you need speed + order -- hashmap + list.
