Reverse Between L and R

🔴 The Problem
Reverse positions left=2 to right=4 in: 1 → 2 → 3 → 4 → 5
🎯 Think of it Like This
Imagine people standing in a line. You want to reverse the order of people at positions 2-4.

Strategy: Take each person from the sub-line and move them to the front of that sub-line, one at a time.

Person at pos 3 → moves in front of pos 2
Person at pos 4 → moves in front of pos 3
Done! The sub-line is reversed.

🧠 Memorize These 3 Lines
The entire reversal loop is just 3 lines repeated. Trace what each does:
// Inside the loop: const nodeToMove = curr.next; // grab next node curr.next = nodeToMove.next; // skip over it nodeToMove.next = beforeLeft.next; // it points to current front beforeLeft.next = nodeToMove; // it becomes the new front
Pattern: detach → point to front → become front
Remember: curr never moves. It stays at node 2 forever. Only the nodes after it get relocated.
💻 Full Code (JavaScript)
function reverseBetween(head, left, right) { const dummy = { next: head }; let beforeLeft = dummy; // Walk to the node before 'left' for (let i = 0; i < left - 1; i++) { beforeLeft = beforeLeft.next; } let curr = beforeLeft.next; // this is the leftNode // Repeat (right - left) times: // grab, detach, insert at front for (let i = 0; i < right - left; i++) { const nodeToMove = curr.next; curr.next = nodeToMove.next; nodeToMove.next = beforeLeft.next; beforeLeft.next = nodeToMove; } return dummy.next; }
Time: O(n)
Space: O(1)
⚠️ Common Mistakes
❌ Moving curr forward — curr stays put! It's always at node 2.
❌ Wrong loop count — it's right - left iterations, not right - left + 1.
❌ Forgetting dummy node — if left=1, there's no node before it. Dummy handles this.
❌ Wrong order of pointer updates — the 3 lines must execute in exactly that order, or you lose references.
💡 Key Insights
1. The "insert at front" trick means you never need a separate reconnection step. The list stays connected throughout.
2. Each iteration: one node "jumps" from behind curr to right after beforeLeft. That's it.
3. If you can trace these 2 frames on paper, you understand the algorithm. Practice drawing it.
