Find Middle Node
LC #876 | Slow/Fast Pointer Application
🔴 The Problem
Given the head of a linked list, return the middle node.
If there are two middle nodes, return the second one.
Example: [1] → [2] → [3] → [4] → [5] → null

Answer: node 3 (the middle)
🧠 Intuition
Use slow (1 step) and fast (2 steps).
When fast reaches the end, slow is at the middle.
Why? Fast travels 2x distance → slow is at half.
📸 Trace — Odd Length [1→2→3→4→5]

init node 1 node 1 —
1 node 2 node 3 no
2 node 3 node 5 fast.next = null → stop
Slow is at node 3 — the middle!

💻 Code (JavaScript)
function middleNode(head) { let slow = head; let fast = head; while (fast !== null && fast.next !== null) { slow = slow.next; // 1 step fast = fast.next.next; // 2 steps } return slow; // middle node }
⚠️ Edge Cases
Single node: fast = head, fast.next = null → loop never runs → return head
Two nodes: after 1 step, fast = null → return slow (2nd node)
Even vs odd: the while condition naturally handles both — fast lands on null (even) or last node (odd)
Time: O(n)
Space: O(1)
💡 Key Insights
1. This is the simplest slow/fast application — pure pattern, no extras.
2. "Second middle" for even length comes free — slow naturally lands there.
3. This becomes a building block: sort list (split at middle), palindrome check (find middle → reverse second half).
🔗 Builds On
Two Pointers: Slow & Fast — this is the direct application. If you know the pattern, this problem is free.
