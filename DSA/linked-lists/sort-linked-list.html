Sort a Linked List
LC #148 | Merge Sort with Slow/Fast Split
🔴 The Problem
Sort a linked list in O(n log n) time and O(1) extra space.

🧠 Strategy — Merge Sort
1. Split the list in half (find middle with slow/fast).
2. Recursively sort each half.
3. Merge the two sorted halves (same as LC #21).
Why Merge Sort?
Quick sort needs random access (bad for linked lists).
Merge sort only needs sequential access + merge — perfect for lists!
Split at middle is O(n) with slow/fast, merge is O(n). Recursion depth is O(log n).

✂️ The Split — Key Detail
We need slow to stop one before the middle so we can cut the link:
Use fast = head.next (not head) so slow stops one step earlier.

[4] → [2] | [1] → [3]
slow at node 2, cut: slow.next = null. Second half starts at old slow.next.
💻 Code (JavaScript)
function sortList(head) { // Base case: empty or single node if (head === null || head.next === null) { return head; } // Split: find middle (stop one before) let slow = head, fast = head.next; while (fast !== null && fast.next !== null) { slow = slow.next; fast = fast.next.next; } const secondHalf = slow.next; slow.next = null; // cut! // Sort each half const left = sortList(head); const right = sortList(secondHalf); // Merge (same as LC #21) return merge(left, right); } function merge(l1, l2) { const dummy = { next: null }; let tail = dummy; while (l1 !== null && l2 !== null) { if (l1.val <= l2.val) { tail.next = l1; l1 = l1.next; } else { tail.next = l2; l2 = l2.next; } tail = tail.next; } tail.next = l1 !== null ? l1 : l2; return dummy.next; }
Time: O(n log n)
Space: O(log n)
Space is O(log n) for recursion stack. True O(1) space needs bottom-up merge sort.
💡 Key Insights
1. This is find middle + merge two sorted lists — two Level 2 patterns glued together with recursion.
2. The subtle fast = head.next trick ensures slow stops before the middle, so we can cut the list cleanly.
3. The merge function is identical to LC #21 — you've already written it!
