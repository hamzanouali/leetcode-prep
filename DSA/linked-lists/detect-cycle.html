🧠 Part 1 — Detect (LC #141)
Slow moves 1 step. Fast moves 2 steps.
If there's a cycle, fast laps slow — they must collide.
If no cycle, fast hits null.
Step Slow Fast Meet?
init node 1 node 1 —
1 node 2 node 3 no
2 node 3 node 5 no
3 node 4 node 4 collision!
They meet at node 4 — cycle confirmed!
🧠 Part 2 — Find Cycle Start (LC #142)
The Math Trick
After collision, reset one pointer to head.
Move both at 1 step. Where they meet = cycle entry.
Why? The distance from head to cycle start = distance from collision point to cycle start (going around).
Step P1 (from head) P2 (from collision) Meet?
init node 1 node 4 —
1 node 2 node 5 no
2 node 3 node 3 meet!
Cycle starts at node 3!
💻 Detect Cycle (LC #141)
function hasCycle(head) { let slow = head; let fast = head; while (fast !== null && fast.next !== null) { slow = slow.next; fast = fast.next.next; if (slow === fast) return true; } return false; }
💻 Find Cycle Start (LC #142)
function detectCycle(head) { let slow = head; let fast = head; while (fast !== null && fast.next !== null) { slow = slow.next; fast = fast.next.next; if (slow === fast) { // Phase 2: find entry point let entry = head; while (entry !== slow) { entry = entry.next; slow = slow.next; } return entry; } } return null; }
📐 Why Phase 2 Works
Let a = distance from head to cycle start.
Let b = distance from cycle start to collision point.
Let c = cycle length.

At collision: slow traveled a + b, fast traveled a + b + c.
Since fast = 2 × slow: a + b + c = 2(a + b) → c = a + b → a = c - b.

So the distance from head to cycle start equals the distance from collision to cycle start (going forward in the cycle).
Time: O(n)
Space: O(1)
💡 Key Insights
1. Phase 1 (detect) uses the same slow/fast template as find middle — just add a collision check.
2. Phase 2 (find start) is a pure math insight — two pointers at speed 1, one from head, one from collision.
3. HashSet approach works too (O(n) space) but Floyd's is O(1) space — interviewers love it.
