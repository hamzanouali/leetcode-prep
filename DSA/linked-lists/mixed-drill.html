Mixed Drill
The Real Interview Flow
Read the problem. Step 1: pick the pattern. Step 2: plan the approach. Step 3: check the code. Reveal each step only after committing to your answer.
Drill 1
Given a linked list, return the node where a cycle begins. If no cycle, return null.
Constraint: O(1) extra space
Step 1: What pattern? ▶
Floyd's Cycle Detection Slow/Fast + Reset
Step 2: Approach ▶
Phase 1: Slow (1 step) and fast (2 steps) until they meet.
Phase 2: Reset one pointer to head. Move both at speed 1. They meet at the cycle entry.
O(1) space rules out a hashset approach.
Step 3: Code ▶
function detectCycle(head) { let slow = head, fast = head; while (fast && fast.next) { slow = slow.next; fast = fast.next.next; if (slow === fast) { slow = head; // reset to head while (slow !== fast) { slow = slow.next; fast = fast.next; } return slow; // cycle start } } return null; }
Drill 2
Given a linked list where nodes have duplicates (sorted), remove ALL nodes that have duplicates. Keep only distinct values.
Example: 1->2->3->3->4->4->5 becomes 1->2->5
Step 1: What pattern? ▶
Dummy Head Pointer Skip
Step 2: Approach ▶
Head could be removed (1->1->2 becomes 2), so dummy head is essential.
Walk with a prev pointer. When you find duplicates, skip ALL of them. prev.next jumps over the entire duplicate group.
Step 3: Code ▶
function deleteDuplicates(head) { const dummy = { next: head }; let prev = dummy; while (prev.next) { let curr = prev.next; while (curr.next && curr.next.val === curr.val) { curr = curr.next; // skip dupes } if (prev.next === curr) { prev = prev.next; // no dupes, advance } else { prev.next = curr.next; // skip ALL dupes } } return dummy.next; }
Drill 3
Given a singly linked list L0 -> L1 -> ... -> Ln, reorder it to L0 -> Ln -> L1 -> Ln-1 -> L2 -> Ln-2 -> ...
Constraint: O(1) extra space, modify in-place
Step 1: What pattern? ▶
Slow/Fast (Find Middle) Reversal Interleave / Merge
Step 2: Approach ▶
Three steps:
1) Find middle with slow/fast
2) Reverse the second half
3) Interleave first half and reversed second half
This is the same skeleton as palindrome check, but merge instead of compare.
Step 3: Code ▶
function reorderList(head) { // 1. Find middle let slow = head, fast = head; while (fast.next && fast.next.next) { slow = slow.next; fast = fast.next.next; } // 2. Reverse second half let prev = null, curr = slow.next; slow.next = null; // cut while (curr) { const next = curr.next; curr.next = prev; prev = curr; curr = next; } // 3. Interleave let l1 = head, l2 = prev; while (l2) { const t1 = l1.next, t2 = l2.next; l1.next = l2; l2.next = t1; l1 = t1; l2 = t2; } }
Drill 4
Given a linked list and a value x, partition it such that all nodes less than x come before nodes greater than or equal to x. Preserve relative order.
Example: 1->4->3->2->5->2 with x=3 becomes 1->2->2->4->3->5
Step 1: What pattern? ▶
Two Dummy Heads Split & Reconnect
Step 2: Approach ▶
Create two dummy heads: one for the "less" list, one for the "greater-or-equal" list.
Walk the original list, appending each node to the right list.
Connect less-tail.next = greater-head. Set greater-tail.next = null (important!).
Step 3: Code ▶
function partition(head, x) { const lessD = { next: null }; const greatD = { next: null }; let l = lessD, g = greatD, curr = head; while (curr) { if (curr.val < x) { l.next = curr; l = l.next; } else { g.next = curr; g = g.next; } curr = curr.next; } g.next = null; // terminate! l.next = greatD.next; // connect return lessD.next; }
Drill 5
Sort a linked list in ascending order.
Constraint: O(n log n) time, O(1) space
Step 1: What pattern? ▶
Merge Sort Slow/Fast (Split) Dummy Head (Merge)
Step 2: Approach ▶
O(n log n) + linked list = merge sort. Quick sort needs random access; merge sort doesn't.
Split: slow/fast to find middle, cut in half.
Recurse: sort each half.
Merge: dummy head + comparison walk (same as LC #21).
Step 3: Code ▶
function sortList(head) { if (!head || !head.next) return head; // Split let slow = head, fast = head.next; while (fast && fast.next) { slow = slow.next; fast = fast.next.next; } const mid = slow.next; slow.next = null; // Sort halves + merge return merge(sortList(head), sortList(mid)); } function merge(l1, l2) { const d = { next: null }; let t = d; while (l1 && l2) { if (l1.val <= l2.val) { t.next = l1; l1 = l1.next; } else { t.next = l2; l2 = l2.next; } t = t.next; } t.next = l1 || l2; return d.next; }
