LRU Cache
LC #146 | Doubly-Linked List + HashMap
🔴 The Problem
Design a data structure with O(1) get and put.
When capacity is exceeded, evict the least recently used item.
get(key)
Return value, mark as recently used. Return -1 if missing.
put(key, val)
Insert/update. If over capacity, evict LRU.
🧠 Data Structure
HashMap: key → node reference (O(1) lookup).
Doubly Linked List: orders by recency.
Head = most recent, Tail = least recent (LRU).
Two dummy sentinels (head & tail) simplify edge cases.

🔧 Two Helper Methods
remove(node) — unlink a node from the DLL. O(1) because we have prev and next pointers.

addToFront(node) — insert right after head dummy. O(1).

Every operation is just remove + addToFront (move to front), or addToFront + possibly remove from back (evict).
💻 Code (JavaScript)
class LRUCache { constructor(capacity) { this.cap = capacity; this.map = new Map(); // Dummy sentinels this.head = { prev: null }; this.tail = { next: null }; this.head.next = this.tail; this.tail.prev = this.head; } _remove(node) { node.prev.next = node.next; node.next.prev = node.prev; } _addToFront(node) { node.next = this.head.next; node.prev = this.head; this.head.next.prev = node; this.head.next = node; } get(key) { if (!this.map.has(key)) return -1; const node = this.map.get(key); this._remove(node); this._addToFront(node); return node.val; } put(key, value) { if (this.map.has(key)) { this._remove(this.map.get(key)); } const node = { key, val: value }; this.map.set(key, node); this._addToFront(node); if (this.map.size > this.cap) { const lru = this.tail.prev; this._remove(lru); this.map.delete(lru.key); } } }
❓ Why Doubly Linked?
Singly linked: removing a node needs its predecessor → O(n) to find.
Doubly linked: node has prev → remove in O(1).
The hashmap gives us the node. DLL lets us remove and reinsert it instantly.
Time: O(1) per op
Space: O(cap)
💡 Key Insights
1. HashMap = O(1) lookup. DLL = O(1) reorder. Together = O(1) everything.
2. Dummy sentinels (head/tail) eliminate all null checks — no edge cases for empty list, single item, etc.
3. Store the key in the DLL node so you can delete from the map when evicting.
4. This is a system design staple — the combo of hashmap + ordered structure appears in many real systems (Redis, Memcached).
