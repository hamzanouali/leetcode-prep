Reverse Between L and R
LC #92 | Partial Reversal in Place
🔴 The Problem
Reverse nodes from position left to right (1-indexed).
Everything before left and after right stays in place.
Input: [1] → [2] → [3] → [4] → [5], left=2, right=4

🧠 Strategy
1. Use dummy node (in case left=1, we reverse from head).
2. Walk to node before position left → call it beforeLeft.
3. Reverse (right - left) links using the standard reversal technique.
4. Reconnect the ends.

This "peels" nodes and inserts them at the front of the reversed section.

💻 Code (JavaScript)
function reverseBetween(head, left, right) { const dummy = { next: head }; let beforeLeft = dummy; // Step 1: reach node before 'left' for (let i = 0; i < left - 1; i++) { beforeLeft = beforeLeft.next; } // Step 2: reverse (right - left) links let curr = beforeLeft.next; for (let i = 0; i < right - left; i++) { const nodeToMove = curr.next; curr.next = nodeToMove.next; nodeToMove.next = beforeLeft.next; beforeLeft.next = nodeToMove; } return dummy.next; }
🔑 The "Insert at Front" Trick
Each iteration does 3 pointer moves:
1. curr.next = nodeToMove.next (detach)
2. nodeToMove.next = beforeLeft.next (point to current front)
3. beforeLeft.next = nodeToMove (make it the new front)

curr never moves! It stays at node 2 the whole time. Only the nodes after it get peeled off and inserted.
Time: O(n)
Space: O(1)
💡 Key Insights
1. This combines dummy head + iterative reversal — two Level 1 patterns.
2. The "insert at front" technique avoids needing to explicitly reconnect after reversal — the links stay connected throughout.
3. This is the core building block for Reverse K-Group (Level 4) — same logic, repeated in chunks.
