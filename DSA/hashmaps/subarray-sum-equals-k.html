Subarray Sum Equals K
LC #560 | Prefix Sum + Hashmap combo
The Problem
Given an integer array nums and an integer k, return the total number of subarrays whose sum equals k.
Input: nums = [1, 1, 1], k = 2
Output: 2 — subarrays [1,1] at index 0..1 and 1..2
Key Intuition
If prefix[j] - prefix[i] = k, then the subarray from i+1 to j sums to k.
prefix[j] - prefix[i] = k ⇒ prefix[i] = prefix[j] - k
So at each position, ask: how many previous prefix sums equal (currentPrefix - k)? A hashmap tracks prefix sum frequencies for O(1) lookup.

Why map.set(0, 1)?
A subarray from index 0 to j has sum = prefix[j+1] - prefix[0] = prefix[j+1] - 0.

We need that "0" prefix already in the map so subarrays starting at the beginning of the array are counted. Without it, we'd miss every subarray that starts at index 0!
Code
function subarraySum(nums, k) { const map = new Map(); map.set(0, 1); // empty prefix let sum = 0, count = 0; for (const num of nums) { sum += num; if (map.has(sum - k)) { count += map.get(sum - k); } map.set(sum, (map.get(sum) || 0) + 1); } return count; }
When to Use
count subarrays with sum = k subarray sum divisible by k negative numbers present can't use sliding window
Sliding window only works when all numbers are positive (window sum is monotonic). With negatives, you need prefix sum + hashmap!
Key Insights
Insight 1
This is the COMBINATION of two techniques: prefix sum + hashmap lookup. Prefix sum converts "subarray sum" into "difference of two values". Hashmap makes finding that difference O(1).
Insight 2
Can't use sliding window when the array has negative numbers. Shrinking the window might increase or decrease the sum unpredictably.
Insight 3
map.set(0, 1) is the critical base case — forgetting it misses every subarray starting at index 0.
Insight 4
We need the count, not just a boolean — that's why we store frequencies, not just existence. The same prefix sum can occur multiple times, each representing a different valid subarray.
Complexity
Time: O(n)
Space: O(n)
