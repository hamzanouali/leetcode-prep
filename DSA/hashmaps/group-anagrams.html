Group Anagrams
LC #49 | Canonical key → hashmap grouping
Problem
Given an array of strings, group the anagrams together. Return the groups in any order.
Input: ["eat","tea","tan","ate","nat","bat"]
Output: [["eat","tea","ate"], ["tan","nat"], ["bat"]]
Intuition
Anagrams have the same letters in different order.
If we sort each string, anagrams become identical.
Use the sorted string as a hashmap key → group words that share the same key.

Hashmap buckets

Code — Sort Key Approach
function groupAnagrams(strs) { const map = new Map(); for (const s of strs) { const key = s.split('').sort().join(''); if (!map.has(key)) map.set(key, []); map.get(key).push(s); } return Array.from(map.values()); }
Alternative — Count Key
Instead of sorting, count each letter's frequency.
Key = "1,0,0,0,1,0,...,1,0,..." (26 counts joined by commas).
O(k) per string instead of O(k log k) — faster for long strings.
function groupAnagrams(strs) { const map = new Map(); for (const s of strs) { const count = new Array(26).fill(0); for (const c of s) count[c.charCodeAt(0) - 97]++; const key = count.join(','); if (!map.has(key)) map.set(key, []); map.get(key).push(s); } return Array.from(map.values()); }
When to Use This Pattern
"group by property" "canonical form" "equivalence classes"
Whenever items need grouping by some shared invariant, transform each item into a canonical key and bucket them in a hashmap.
Key Insights
Insight 1
The core pattern: transform each item into a canonical key, then group by that key using a hashmap.
Insight 2
Sort key costs O(k log k) per string. Count key costs O(k) per string. Choose based on string length.
Insight 3
This "canonical key grouping" pattern appears in many problems beyond anagrams — group shifted strings, isomorphic strings, etc.
Complexity
Time: O(n · k log k)
Space: O(n · k)
n = number of strings, k = max string length. With count key: O(n · k) time.
