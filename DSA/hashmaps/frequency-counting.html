App 1: Valid Anagram (LC #242)

Problem: Given two strings s and t, return true if t is an anagram of s.

s = "anagram", t = "nagaram"

Step 1: Build frequency map for s

Count every character in s:

a:3
n:1
g:1
r:1
m:1
Step 2: Subtract counts for t

For each char in t, decrement count. If any count goes below zero (or key missing), not an anagram.

a:0
n:0
g:0
r:0
m:0

All zeros! The strings are anagrams.

Code: Valid Anagram
function isAnagram(s, t) { if (s.length !== t.length) return false; const count = {}; for (const c of s) count[c] = (count[c] || 0) + 1; for (const c of t) { if (!count[c]) return false; count[c]--; } return true; }
App 2: Contains Duplicate (LC #217)

Problem: Given an integer array nums, return true if any value appears at least twice.

Approach: Use a Set

Walk through array. For each element, check if it's already in the Set. If yes, return true. Otherwise, add it.

A Set is simpler than a map when you only need to ask: "Have I seen this before?"

Code: Contains Duplicate
function containsDuplicate(nums) { const seen = new Set(); for (const num of nums) { if (seen.has(num)) return true; seen.add(num); } return false; }
When to Use
count occurrences find duplicates anagram permutation check ransom note
Key Insights
1. Frequency Map

{element → count}. Increment for one string, decrement for another. If all counts are zero, the two collections match.

2. Set = "Seen Before?"

When you don't need counts, a Set is simpler. Just check membership: seen.has(x).

3. Fixed Alphabet Trick

For lowercase letters only, an array of size 26 works and is faster than a map. Index = charCodeAt(0) - 97.

Complexity
Time: O(n) Space: O(k)

where k = number of unique elements
