Longest Consecutive Sequence
LC #128 | HashSet + smart starting points
The Problem
Given an unsorted array of integers, find the length of the longest consecutive elements sequence.
Must run in O(n) time — so you can't sort.
Input: [100, 4, 200, 1, 3, 2] → Output: 4  (sequence: 1, 2, 3, 4)
The Input

green = consecutive chain  |  orange = lone numbers
The Intuition
1. Put everything in a Set for O(1) lookup.
2. For each number, check: is (num - 1) in the set?
    If NO → it's the START of a sequence. Count forward.
    If YES → skip it (not a start, someone else will count through it).

The Code
function longestConsecutive(nums) { const set = new Set(nums); let maxLen = 0; for (const num of set) { if (!set.has(num - 1)) { // only start from sequence beginnings let current = num; let len = 1; while (set.has(current + 1)) { current++; len++; } maxLen = Math.max(maxLen, len); } } return maxLen; }
When To Use
"consecutive sequence" "longest streak" "O(n) required so can't sort"
Time: O(n)
Space: O(n)
Key Insights
1. The key trick: only start counting from sequence BEGINNINGS (num-1 not in set). This ensures each element is visited at most twice → O(n).
2. Without the "is this a start?" check, you'd re-count subsequences → O(n²).
3. HashSet gives O(1) lookup — essential for the O(n) guarantee.
