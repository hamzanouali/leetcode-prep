Top K Frequent Elements
LC #347 | Frequency Map + Bucket Sort
The Problem
Given an integer array nums and an integer k, return the k most frequent elements. You may return the answer in any order.
Input: nums = [1,1,1,2,2,3], k = 2
Output: [1, 2]
Intuition
Count frequencies first. Then instead of sorting (O(n log n)), use BUCKET SORT — bucket index = frequency, bucket contents = elements with that frequency. Iterate from the highest bucket down.

Step 3: Iterate from Highest Bucket Down
Walk from bucket[6] down, collecting elements until we have k = 2:
bucket[3] → grab 1   (result: [1])
bucket[2] → grab 2   (result: [1, 2])
Done! result.length = k
Why Bucket Sort?
Regular sort: O(n log n)
Bucket sort: O(n)

Max frequency is at most n, so the bucket array size is bounded by n — no comparison sort needed!
Code Template
function topKFrequent(nums, k) { // Step 1: Count frequencies const freq = new Map(); for (const num of nums) { freq.set(num, (freq.get(num) || 0) + 1); } // Step 2: Bucket sort // bucket[i] = elements with frequency i const buckets = new Array(nums.length + 1) .fill(null).map(() => []); for (const [num, count] of freq) { buckets[count].push(num); } // Step 3: Collect top k const result = []; for (let i = buckets.length - 1; i >= 0 && result.length < k; i--) { result.push(...buckets[i]); } return result; }
When to Use
"top k" "most frequent" "k-th most common"
Key Insights
1. O(n) without comparison sort
Frequency map + bucket sort = O(n). Bucket index is the frequency — natural ordering without comparison sort.
2. Two-layer hashmap pattern
First map values → counts, then organize counts → values. This inversion is the core trick.
3. Heap alternative
A min-heap of size k gives O(n log k) — useful when k « n, but bucket sort is simpler and faster for general case.
4. Bounded bucket array
Max frequency is at most n, so bucket array never exceeds input size. No extra log factor needed.
Complexity
Time: O(n) Space: O(n)
