Two Pointers — Opposite Direction
LC #167, #125, #11  |  The most fundamental array technique
The Pattern
Problem shape: you have a sorted array (or a sequence you can scan from both ends) and need to find a pair or shrink a window.

Idea: place one pointer at the left end and one at the right end. At each step, compare a condition and move one pointer inward. They walk toward each other until they meet.

Decision Logic
At each step, compute the result of nums[L] and nums[R]:

• If sum == target → done!
• If sum < target → need bigger → L++ (move left pointer right)
• If sum > target → need smaller → R-- (move right pointer left)

This works because the array is sorted — moving L right increases the sum, moving R left decreases it.
Code Template
function twoSumSorted(nums, target) { let left = 0, right = nums.length - 1; while (left < right) { const sum = nums[left] + nums[right]; if (sum === target) return [left, right]; if (sum < target) left++; else right--; } return [-1, -1]; }

Key Insights
Why it works
The array is sorted — moving L right increases the sum, moving R left decreases it. So you can always steer toward the target. You never skip a valid pair because you only eliminate values that are provably too big or too small.
Complexity win
O(n) instead of brute-force O(n²). Both pointers together visit each element at most once. Left only moves right, right only moves left — total moves ≤ n.
The core invariant
At every step, the answer (if it exists) is always between L and R. Each move rules out one element from consideration — without losing any valid candidates.
