3Sum
LC #15 | Sort + Fix one + Two Pointers
The Problem

Given an integer array nums, find all unique triplets [nums[i], nums[j], nums[k]] such that i != j != k and nums[i] + nums[j] + nums[k] == 0.

No duplicate triplets in the result.

Intuition

Sort the array first. Then for each element i, use two pointers (L, R) on the remaining subarray to find pairs that sum to -nums[i].

Skip duplicates at every level — both the outer loop (i) and inner pointers (L, R) — to avoid repeated triplets.

Code Template
function threeSum(nums) { nums.sort((a, b) => a - b); const result = []; for (let i = 0; i < nums.length - 2; i++) { // skip duplicate i if (i > 0 && nums[i] === nums[i - 1]) continue; let left = i + 1, right = nums.length - 1; while (left < right) { const sum = nums[i] + nums[left] + nums[right]; if (sum === 0) { result.push([nums[i], nums[left], nums[right]]); // skip duplicate L and R while (left < right && nums[left] === nums[left + 1]) left++; while (left < right && nums[right] === nums[right - 1]) right--; left++; right--; } else if (sum < 0) left++; else right--; } } return result; }
When to Use
"find triplets" "sum to target" "unique combinations" "quadruplets (4Sum)"
Key Insights
Sort First

Sorting enables two-pointer scanning and makes duplicate-skipping trivial (identical values are adjacent).

Skip Duplicates at BOTH Levels

Outer loop: if nums[i] === nums[i-1], skip. Inner loop: after a match, advance L past duplicates and retreat R past duplicates.

O(n^3) to O(n^2)

Brute force checks all triples in O(n^3). Fix one + two pointers reduces to O(n^2).

Extends to kSum

Same pattern works for 4Sum (add one more outer loop) and generalises to kSum with k-2 nested loops + two pointers.
