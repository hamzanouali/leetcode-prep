Container With Most Water
LC #11 | Two Pointers — Greedy Shrink
The Problem
Given an array height[] of n lines, find two lines that together with the x-axis form a container holding the most water.

area = min(height[L], height[R]) * (R - L)

Intuition
Start wide with L=0 and R=end (maximum width).

Always move the shorter side inward. Why?
• The shorter line is the bottleneck — it caps the water height.
• Moving the taller side can only decrease or maintain height while the width always shrinks.
• Moving the shorter side might find a taller line → potential improvement.
Step-by-Step Trace
Step 1
L=0 (h=1) R=8 (h=7): area = min(1,7) × 8 = 8
height[L] < height[R] → move L++
Step 2
L=1 (h=8) R=8 (h=7): area = min(8,7) × 7 = 49
height[L] ≥ height[R] → move R--
Step 3
L=1 (h=8) R=7 (h=3): area = min(8,3) × 6 = 18
height[L] ≥ height[R] → move R--
Step 4
L=1 (h=8) R=6 (h=8): area = min(8,8) × 5 = 40
height[L] ≥ height[R] → move R--
...continues until L meets R. max = 49
Code
function maxArea(height) { let left = 0, right = height.length - 1; let max = 0; while (left < right) { const area = Math.min(height[left], height[right]) * (right - left); max = Math.max(max, area); if (height[left] < height[right]) { left++; // move shorter side } else { right--; // move shorter side } } return max; }
When to Use
maximize area / container two pointer greedy sorted-ends shrink
Key Insights
1. Shorter side is the bottleneck
The water level is capped by the shorter line. Moving it is the only way to potentially raise the height and find a larger area.
2. Width always decreases
Every step shrinks width by 1. So we need the height to increase to beat the current max — only possible by moving the shorter side.
3. Greedy never misses optimal
Moving the taller side can't help — the min height stays the same or drops, while width shrinks. So we safely skip all those pairs.
Complexity
Time: O(n) Space: O(1)
