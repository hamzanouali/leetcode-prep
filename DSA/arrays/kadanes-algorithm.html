Kadane's Algorithm
LC #53 | Maximum Subarray Sum in O(n)
The Problem
Given an integer array nums, find the contiguous subarray (containing at least one element) which has the largest sum, and return that sum.

Answer: subarray [4, -1, 2, 1] → sum = 6
Core Intuition
The Decision at Each Element
At index i, you have exactly two choices:
1. START a new subarray beginning here
2. EXTEND the current subarray by adding this element
The Key Formula
currentSum = max(nums[i], currentSum + nums[i])
Translation: Is the past helping or hurting?
If currentSum < 0, the past is dragging us down — drop it and restart.
Step-by-Step Trace
Tracking currentSum and maxSum through the array:
i nums[i] curr + nums[i] currentSum maxSum
0 -2 — -2 -2
1 1 -2+1 = -1 1 1
2 -3 1-3 = -2 -2 1
3 4 -2+4 = 2 4 4
4 -1 4-1 = 3 3 4
5 2 3+2 = 5 5 5
6 1 5+1 = 6 6 6
7 -5 6-5 = 1 1 6
8 4 1+4 = 5 5 6
Final answer: maxSum = 6
Code (JavaScript)
function maxSubArray(nums) { let currentSum = nums[0]; let maxSum = nums[0]; for (let i = 1; i < nums.length; i++) { // Start fresh or extend? currentSum = Math.max(nums[i], currentSum + nums[i]); maxSum = Math.max(maxSum, currentSum); } return maxSum; }

Key Insights
Insight #1
The decision at each step: is the past helping or hurting? If currentSum < 0, the accumulated prefix is a net negative — drop it entirely and restart from the current element.
Insight #2
This is actually dynamic programming. Define dp[i] = max subarray sum ending at index i. Then dp[i] = max(nums[i], dp[i-1] + nums[i]). Kadane's just optimizes space from O(n) to O(1) by only keeping the previous value.
Insight #3
Variation: to track the actual subarray start/end indices, record the start index whenever you restart (choose nums[i] over currentSum + nums[i]), and record the end index whenever maxSum updates.
Complexity
Time: O(n) Space: O(1)
