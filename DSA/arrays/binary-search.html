Binary Search
LC #704, #35, #34  |  Halve the search space every step
The Problem
Given a sorted array, find a target value's index — or determine where it would be inserted.
Brute force scans every element: O(n). Can we do better?
The Idea
Since the array is sorted, compare the target to the middle element:
• If equal → found it!
• If target is bigger → search the right half
• If target is smaller → search the left half
Each step eliminates half the remaining elements.

Code Template
function binarySearch(nums, target) { let left = 0, right = nums.length - 1; while (left <= right) { const mid = Math.floor((left + right) / 2); if (nums[mid] === target) return mid; if (nums[mid] < target) left = mid + 1; else right = mid - 1; } return -1; // not found (or return left for insert pos) }
Variations
Left Bound (First Occurrence)
function leftBound(nums, target) { let left = 0, right = nums.length - 1; while (left <= right) { const mid = Math.floor((left + right) / 2); if (nums[mid] >= target) right = mid - 1; else left = mid + 1; } return left; // first index where nums[i] >= target }
When we find target, don't stop — keep searching left by setting R = M - 1.
Right Bound (Last Occurrence)
function rightBound(nums, target) { let left = 0, right = nums.length - 1; while (left <= right) { const mid = Math.floor((left + right) / 2); if (nums[mid] <= target) left = mid + 1; else right = mid - 1; } return right; // last index where nums[i] <= target }
When we find target, don't stop — keep searching right by setting L = M + 1.
When to Use
Sorted array "Search for..." "Find minimum that satisfies" Monotonic condition Insert position First/last occurrence
Key Insights
1. Half the work every step
Each comparison eliminates half the search space. That's why it's O(log n) — even 1 billion elements only takes ~30 steps.
2. left <= right vs left < right
Using <= means you check every element (search space is [L, R] inclusive). Using < stops one step early — useful when you need L and R to converge to a single answer without overshooting.
3. "Binary search the answer"
Sometimes you can't search the array directly. Instead, binary search on the answer space: guess a value, check if it works, then narrow the range. Works whenever the condition is monotonic (once true, stays true).
