Prefix Sum
LC #303, #560, #238 | Precompute to answer range queries in O(1)
The Problem
Given an array of numbers, how do you get the sum of any subarray in O(1) time?
Brute force: loop from i to j and add up — that's O(n) per query. With many queries, this is too slow.
The Idea
Precompute a prefix sum array where each entry stores the cumulative sum of all elements before it. Then any range sum is just one subtraction.

Range Sum Formula
sum(i, j) = prefix[j + 1] − prefix[i]

Code Template
Build the Prefix Array
function buildPrefix(nums) { const prefix = [0]; for (let i = 0; i < nums.length; i++) { prefix.push(prefix[i] + nums[i]); } return prefix; }
Query Any Range in O(1)
function rangeSum(prefix, i, j) { return prefix[j + 1] - prefix[i]; }
Usage
const nums = [2, 4, 1, 3, 5]; const prefix = buildPrefix(nums); // prefix = [0, 2, 6, 7, 10, 15] rangeSum(prefix, 1, 3); // 10 - 2 = 8 rangeSum(prefix, 0, 4); // 15 - 0 = 15 rangeSum(prefix, 2, 2); // 7 - 6 = 1
When to Use
range sum query subarray sum multiple sum queries product except self subarray sum = k contiguous sum
Key Insights
1. What prefix[i] means
prefix[i] = sum of the first i elements (indices 0 to i-1). That's why prefix has length n + 1 and starts with 0.
2. Any range sum = one subtraction
Instead of looping through elements, just compute prefix[j+1] - prefix[i]. Two lookups, one subtraction — O(1).
3. Build once, query forever
Building the prefix array is O(n). After that, every query is O(1). With Q queries, total is O(n + Q) instead of O(n * Q).
4. Variations work the same way
Prefix XOR, prefix product, and prefix count all follow the same pattern — precompute cumulative values, then extract any range with one operation.
Complexity
Build: O(n) time
Query: O(1) time
Space: O(n)
